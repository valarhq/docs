[OpenCode](https://opencode.ai) is an open-source terminal coding agent. It talks to any backend that speaks the OpenAI-compatible protocol, and Valar does — so connecting them is purely a matter of adding one provider entry to your OpenCode config.

## How the pieces fit

OpenCode reads a JSONC config file and routes model requests to whichever provider you register. You register Valar under the key `valar`, point it at `https://api.valarhq.ai/v1`, and list the models you want to use. From then on you reference any Valar model as `valar/<model-id>` — for example `valar/zai-org/GLM-5.1-FP8`.

## Get set up

<Steps>
  <Step title="Install OpenCode">
    Follow the instructions at [opencode.ai](https://opencode.ai). The one-line installer covers most setups:

    ```bash
    curl -fsSL https://opencode.ai/install | bash
    ```
  </Step>
  <Step title="Create a Valar API key">
    Sign in at [Valar](https://app.valarhq.ai) and generate an API key. Keep it handy for the next step.
  </Step>
  <Step title="Create the config directory">
    ```bash
    mkdir -p ~/.config/opencode
    ```
  </Step>
  <Step title="Write the provider config">
    Save the block below to `~/.config/opencode/opencode.jsonc`, replacing `YOUR_VALAR_API_KEY` with the key you generated.
  </Step>
</Steps>

```jsonc
{
  "$schema": "https://opencode.ai/config.json",
  "model": "valar/zai-org/GLM-5.1-FP8", // primary model for coding work
  "small_model": "valar/openai/gpt-oss-120b", // cheaper model for light tasks
  "provider": {
    "valar": {
      "npm": "@ai-sdk/openai-compatible",
      "options": {
        "baseURL": "https://api.valarhq.ai/v1",
        "apiKey": "YOUR_VALAR_API_KEY",
      },
      "models": {
        "zai-org/GLM-5.1-FP8": { "name": "GLM-5.1" },
        "openai/gpt-oss-120b": { "name": "gpt-oss-120b" },
        "MiniMaxAI/MiniMax-M2.7": { "name": "MiniMax-M2.7" },
        "google/gemma-4-31B-it": { "name": "Gemma 4 31B IT" },
      },
    },
  },
}
```

<Note>
  The `model` and `small_model` keys set the defaults OpenCode reaches for. Point each at a model you've listed under `provider.valar.models`, prefixed with `valar/`.
</Note>

## Start coding

Launch OpenCode from your project directory:

```bash
opencode
```

Once it's running, type `/models`, search for "Valar", and switch between any of the models you defined. OpenCode will route each request through the `valar` provider to Valar.

<Accordion title="Adding a model later">
  Whenever Valar ships a model you want to use, add one line under `provider.valar.models` with its model ID and a display name, then reference it as `valar/<model-id>`:

  ```jsonc
  "models": {
    "zai-org/GLM-5.1-FP8": { "name": "GLM-5.1" },
    "moonshotai/Kimi-K2.6": { "name": "Kimi-K2.6" },
  },
  ```

  See [Models](/models) for the current list of model IDs.
</Accordion>

For more on what OpenCode can do once it's connected, see its [usage guide](https://opencode.ai/docs/#usage).
