---
title: "Webhooks"
description: "Receive a callback when a background request finishes instead of polling for it"
---

When a request can take a while to finish, you don't have to poll `GET /v1/responses/{response_id}` in a loop. Attach a callback URL to the request and Valar delivers the finished result to your server as an HTTP POST. The feature works on `POST /v1/responses` and `POST /v1/chat/completions`, and is configured entirely through two `metadata` keys.

## Parameters

Both keys live inside the `metadata` object on the create request.

<ParamField path="metadata.completion_webhook" type="string">
  The destination URL for the callback. Must be an `https` URL that resolves to a public address — Valar refuses `http`, private, and loopback targets, and never follows redirects. If you omit it or pass a value that can't be used, no callback is sent and the request itself is unaffected.
</ParamField>

<ParamField path="metadata.webhook_token" type="string">
  An optional shared secret. Valar sends its value as a Bearer token in the callback's `Authorization` header, and uses it as the key for the `X-Valar-Signature` HMAC — so your endpoint can confirm the request is genuine and verify it hasn't been tampered with or replayed. See [Verify the signature](#verify-the-signature).
</ParamField>

## Set up an endpoint

<Steps>
  <Step title="Expose a route that accepts POST">
    Your endpoint receives a JSON body and should respond as soon as it has accepted the payload. The body matches exactly what `GET /v1/responses/{response_id}` returns for the same request.

    ```python
    from fastapi import FastAPI, Request
    
    app = FastAPI()
    
    @app.post("/hooks/inference-done")
    async def inference_done(request: Request):
        payload = await request.json()
        # payload is the same object GET /v1/responses/{id} returns
        enqueue_for_processing(payload["id"], payload)
        return {"ok": True}
    ```
  </Step>
  <Step title="Verify the Authorization header">
    If you set `webhook_token`, reject any incoming call whose header doesn't match. Compare against `Authorization: Bearer <your token>` and return a 4xx for anything else.

    ```python
    from fastapi import Header, HTTPException
    
    WEBHOOK_TOKEN = "whk_3f0a9c2e7b14"
    
    def assert_authorized(authorization: str = Header(default="")):
        if authorization != f"Bearer {WEBHOOK_TOKEN}":
            raise HTTPException(status_code=401)
    ```
  </Step>
  <Step title="Submit a request that points at your endpoint">
    Set `background=True` and add the callback URL (and token, if you use one) to `metadata`.

    <CodeGroup>

    ```python File 1
    from openai import OpenAI
    
    client = OpenAI(
        base_url="https://api.valarhq.ai/v1", # or read OPENAI_BASE_URL from the environment
        api_key="YOUR_VALAR_API_KEY",  # or read OPENAI_API_KEY from the environment
    )
    
    response = client.responses.create(
        model="zai-org/GLM-5.1-FP8",
        input="Summarize this document.",
        background=True,
        metadata={
            "completion_webhook": "https://app.example.com/hooks/inference-done",
            "webhook_token": "whk_3f0a9c2e7b14",
        },
    )
    ```

    </CodeGroup>
  </Step>
</Steps>

## What Valar sends

The callback is a single POST request.

<ResponseField name="method" type="POST">
  Always a POST to the URL in `completion_webhook`.
</ResponseField>

<ResponseField name="Content-Type" type="header">
  `application/json`.
</ResponseField>

<ResponseField name="Authorization" type="header">
  Present only when `webhook_token` was set. Carries `Bearer <webhook_token>`.
</ResponseField>

<ResponseField name="X-Valar-Response-Id" type="header">
  The response `id`, also present in the body. It's your deduplication key and an input to signature verification.
</ResponseField>

<ResponseField name="X-Valar-Signature" type="header">
  Present only when `webhook_token` was set. A Stripe-style `t=<unix>,v1=<hmac>` value you can verify to confirm integrity and reject replays. See [Verify the signature](#verify-the-signature).
</ResponseField>

<ResponseField name="body" type="object">
  The full response object, byte-for-byte identical to a `GET /v1/responses/{response_id}` call. The response `id` lives here and is your key for deduplication.
</ResponseField>

## Verify the signature

When you set a `webhook_token`, every callback carries an `X-Valar-Signature` header so you can confirm the payload came from Valar, hasn't been altered, and isn't a replay. The header is Stripe-style:

```
X-Valar-Signature: t=1717000000,v1=3f8a...c1
```

- `t` is the Unix timestamp (seconds) when Valar signed the delivery.
- `v1` is `HMAC-SHA256`, keyed by your `webhook_token`, over the string `{t}.{response_id}.{raw_body}` — the timestamp, the `X-Valar-Response-Id` header, and the exact raw request body, joined with `.`.

<Steps>
  <Step title="Recompute the HMAC">
    Build the signed payload from `t`, the `X-Valar-Response-Id` header, and the **raw** request body, then HMAC it with your `webhook_token`.
  </Step>
  <Step title="Compare in constant time">
    Reject the request if your computed value doesn't equal `v1`. Use a constant-time comparison.
  </Step>
  <Step title="Reject stale timestamps">
    Reject the request if `t` is outside a tolerance window — 300 seconds is a good default. This is what stops a captured delivery from being replayed later.
  </Step>
</Steps>

```python
import hashlib, hmac, time
from fastapi import Request, HTTPException

WEBHOOK_TOKEN = "whk_3f0a9c2e7b14"
TOLERANCE_SECONDS = 300

async def verify(request: Request):
    raw = await request.body()  # the exact bytes — do not re-serialize the JSON
    parts = dict(p.split("=", 1) for p in request.headers["X-Valar-Signature"].split(","))
    t, v1 = parts["t"], parts["v1"]

    if abs(time.time() - int(t)) > TOLERANCE_SECONDS:
        raise HTTPException(400, "stale signature")

    response_id = request.headers["X-Valar-Response-Id"]
    signed = f"{t}.{response_id}.".encode() + raw
    expected = hmac.new(WEBHOOK_TOKEN.encode(), signed, hashlib.sha256).hexdigest()
    if not hmac.compare_digest(expected, v1):
        raise HTTPException(401, "bad signature")
```

<Note>
  Verify against the **raw** request bytes, exactly as received. Re-serializing the parsed JSON can change key order or whitespace and break the comparison.
</Note>

<Tip>
  The signature covers integrity and replay; the `Authorization: Bearer` header covers identity. Checking the signature is the stronger of the two — a valid signature already proves the sender holds your `webhook_token`.
</Tip>

## Delivery semantics

<Warning>
  The same callback can be delivered more than once. Treat the response `id` in the body as an idempotency key and ignore any `id` you have already processed.
</Warning>

- **Retries.** A delivery that returns a non-2xx status or hits a network error is retried up to **3 times**. Return a **2xx** as soon as you accept the payload to stop the retries.
- **Timeout.** Each attempt has a **30 second** ceiling. A timeout counts as a failed attempt and triggers the next retry.
- **Best-effort.** Callbacks are best-effort. A failure is logged but never touches the response record or the API, and the result stays available through `GET /v1/responses/{response_id}` regardless of whether delivery ever succeeded.

  <Step title="Run a listener that prints the body and returns 200">
    ```bash
    python -c "
    from http.server import HTTPServer, BaseHTTPRequestHandler; import json
    class H(BaseHTTPRequestHandler):
     def do_POST(self):
      print(json.dumps(json.loads(self.rfile.read(int(self.headers['Content-Length']))), indent=2))
      self.send_response(200); self.end_headers()
    HTTPServer(('127.0.0.1', 8765), H).serve_forever()
    "
    ```
