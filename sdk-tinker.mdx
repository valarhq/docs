The `valar.tinker` helpers let a
[Tinker](https://tinker-docs.thinkingmachines.ai/) RL or training loop sample
from ValarAI-hosted models. They map Tinker's token-level sampling interface
onto ValarAI's raw-token Responses path, optionally apply a LoRA adapter, and
stream logprobs back to your trainer.

<Note>
  These helpers need `tinker-cookbook` installed next to `valar-sdk`.
  Constructing a `ValarTokenCompleter` without `tinker-cookbook` available
  raises [`valar.InferenceError`](/traces-sdk-errors).
</Note>

## Integration flow

<Steps>
  <Step title="Build a completer">
    Create a `valar.ValarTokenCompleter` from a model id and sampling settings.
    This is the `TokenCompleter` your Tinker loop will call.

    ```python theme={null}
    import valar

    completer = valar.ValarTokenCompleter(
        model="meta-llama/Llama-3.1-8B-Instruct",
        max_tokens=256,
        temperature=0.7,
    )
    ```
  </Step>

  <Step title="Sample on tokenized prompts">
    `await` the completer on a tokenized prompt. You get back the sampled
    tokens, their logprobs, and a stop reason.

    ```python theme={null}
    result = await completer(model_input)
    print(result.tokens)         # list[int] of sampled token ids
    print(result.maybe_logprobs) # list[float] | None
    print(result.stop_reason)    # e.g. "stop", "length"
    ```
  </Step>

  <Step title="Attach a trained adapter (optional)">
    To sample against a Tinker checkpoint instead of a registered LoRA, resolve
    the checkpoint to a signed URL and pass it with an adapter config.

    ```python theme={null}
    import valar

    signed_url = valar.get_tinker_checkpoint_signed_url(service_client, tinker_path)

    completer = valar.ValarTokenCompleter(
        model="meta-llama/Llama-3.1-8B-Instruct",
        max_tokens=256,
        tinker_lora_signed_url=signed_url,
        adapter_config={"r": 16, "alpha": 32},
    )
    ```
  </Step>
</Steps>

## valar.ValarTokenCompleter

A Tinker `TokenCompleter` backed by ValarAI's raw-token Responses API.

```python theme={null}
ValarTokenCompleter(
    *,
    model: str,
    max_tokens: int,
    temperature: float = 1.0,
    top_p: float = 1.0,
    lora: str | None = None,
    tinker_lora_signed_url: str | None = None,
    adapter_config: Mapping[str, Any] | str | None = None,
    tinker_lora_name: str | None = None,
    metadata: Mapping[str, str] | None = None,
    timeout: float | None = None,
    trace: valar.Trace | None = None,
    request_logprobs: bool = True,
)
```

| Parameter | Description |
| --------- | ----------- |
| `model` | **Required.** Model id to sample from. Empty values raise `ValueError`. |
| `max_tokens` | **Required.** Token budget for generation; must be `> 0`, or `ValueError` is raised. |
| `temperature` | Sampling temperature. Default `1.0`. |
| `top_p` | Nucleus sampling cutoff. Default `1.0`. |
| `lora` | A ValarAI-registered LoRA adapter to apply. Cannot be combined with `tinker_lora_signed_url`. |
| `tinker_lora_signed_url` | Signed URL to a Tinker checkpoint archive (see [`get_tinker_checkpoint_signed_url`](#get-tinker-checkpoint-signed-url)). Requires `adapter_config`. Cannot be combined with `lora`. |
| `adapter_config` | LoRA adapter config as a mapping or JSON string. **Required** when `tinker_lora_signed_url` is set. |
| `tinker_lora_name` | Optional human-readable name for the Tinker LoRA. |
| `metadata` | Extra string metadata sent on every request. |
| `timeout` | Per-request timeout in seconds. |
| `trace` | A [`valar.Trace`](/traces-sdk) that attributes requests to a trace. |
| `request_logprobs` | Whether to ask the server for logprobs. Default `True`. |

Supplying both `lora` and `tinker_lora_signed_url`, or setting
`tinker_lora_signed_url` without `adapter_config`, raises `ValueError`.

### Calling the completer

```python theme={null}
async def __call__(model_input, stop=None) -> TokensWithLogprobs
```

`model_input` must expose a callable `.to_ints()` returning the prompt token ids
(Tinker's `ModelInput`). A non-callable `to_ints`, a non-integer token, or an
empty prompt raises `TypeError` or `ValueError`.

`stop` is an optional stop condition: an `int` is wrapped into a single-element
list, a tuple becomes a list, and anything else is passed through unchanged.

The result is a Tinker `TokensWithLogprobs`:

| Field | Description |
| ----- | ----------- |
| `tokens` | `list[int]` — the sampled token ids. |
| `maybe_logprobs` | `list[float] \| None` — per-token logprobs, or `None` when the response carried none. |
| `stop_reason` | The model's stop reason (such as `"stop"` or `"length"`), falling back to the response `status` or `"unknown"`. |

A malformed ValarAI response — missing `output_text`, missing or non-integer
`token_ids`, or mismatched token and logprob lengths — raises
[`valar.InferenceError`](/traces-sdk-errors) with the offending response
attached as `exc.response`.

## get_tinker_checkpoint_signed_url

```python theme={null}
get_tinker_checkpoint_signed_url(service_client, tinker_path: str) -> str
```

Resolves a Tinker checkpoint path into a signed archive URL suitable for the
`tinker_lora_signed_url` argument above. It calls the Tinker service client's
REST client (`get_checkpoint_archive_url_from_tinker_path`) and reads the URL
out of the result — accepting either a plain string or an object or mapping that
carries `url`, `signed_url`, `archive_url`, or `checkpoint_archive_url`.

| Parameter | Description |
| --------- | ----------- |
| `service_client` | A Tinker `ServiceClient` exposing `create_rest_client()`. |
| `tinker_path` | The Tinker checkpoint path to resolve. |

When no URL can be found in the response,
[`valar.InferenceError`](/traces-sdk-errors) is raised.
